# 音视频转文本Android应用 - 开发规划

## 项目目标
开发一个本地运行的Android应用，实现语音/视频转文本功能，支持大模型总结，无需依赖外部服务。

## 技术选型

### 1. 语音识别引擎
**选择：Vosk (离线语音识别)**
- 完全离线运行，符合用户需求
- 支持中文模型
- 模型大小适中（中文小模型约50MB）
- 开源免费

**备选方案：Whisper.cpp**
- 效果更好但模型较大（基础模型约150MB）
- 如Vosk效果不佳可切换

### 2. 音频提取
**选择：Android MediaExtractor + MediaCodec**
- 系统原生API，无需额外依赖
- 轻量级，适合移动设备
- 支持常见音视频格式

### 3. 大模型API
**选择：OkHttp + Gson**
- 支持OpenAI兼容API
- 轻量级HTTP客户端
- 易于扩展

### 4. UI框架
**选择：Material Design + ViewBinding**
- 现代化UI设计
- 良好的用户体验
- 系统原生支持

## 功能模块设计

### 模块1：文件选择
- 使用Storage Access Framework (SAF)
- 支持常见音视频格式：mp3, wav, m4a, mp4, mkv, avi等
- 显示文件信息（名称、大小、时长）

### 模块2：音频处理
- 从视频中提取音频
- 音频格式转换（转为WAV 16kHz，Vosk要求）
- 音频分段处理（长音频分块识别）

### 模块3：语音识别
- 加载Vosk模型
- 实时转写音频
- 显示转写进度
- 支持取消操作

### 模块4：大模型总结
- 配置界面：输入API URL和Token
- 发送转写文本到API
- 流式显示总结结果
- 错误处理和重试

### 模块5：结果管理
- 显示转写文本和总结
- 复制到剪贴板
- 保存为文本文件
- 分享功能

## 注意事项

1. **Vosk模型下载**：首次使用需要下载中文模型，约50MB
2. **内存管理**：长音频处理需要注意内存，使用流式处理
3. **电量消耗**：本地语音识别会消耗较多电量，需要提示用户
4. **错误处理**：完善的错误提示和异常处理
5. **后台处理**：长时间转写需要在后台服务中进行

## 风险与应对

| 风险 | 影响 | 应对措施 |
|------|------|----------|
| Vosk识别准确率不高 | 用户体验差 | 提供Whisper备选方案 |
| 长音频处理内存溢出 | 应用崩溃 | 分段处理，及时释放资源 |
| 大模型API调用失败 | 无法总结 | 完善错误处理，支持重试 |
| 模型下载失败 | 无法使用 | 提供离线模型包，支持手动导入 |
